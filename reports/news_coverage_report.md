## 10. 自动保护: 覆盖率门控 (Coverage Gate)

### 10.1 实现机制

当资讯覆盖率不足时，`core/news_coverage_gate.py` 自动介入:

```
FusionEngine.evaluate(symbol)
       ↓
  get_news_weight_multiplier()
       ↓
  ┌──────────────────────────────┐
  │ coverage ≥ 20%  → ×1.0  正常 │
  │ coverage ≥ 5%   → ×0.3  降权 │
  │ coverage < 5%   → ×0.0  禁用 │
  └──────────────────────────────┘
       ↓
  msg_weight = 0.25 × multiplier
  多余权重 → 资金面 + 趋势面 (各50%)
```

|------|-----|

| 保护级别 | **degraded** |

### 10.3 配置

```yaml
# config/config.yaml
news_data:
  coverage_gate:
    enabled: true
    healthy_threshold: 0.20    # ≥20% → 资讯因子权重 100%

```

## 11. 白名单合规资讯采集器方案

### 11.1 设计原则

| 原则 | 说明 |
|------|------|
| **API/RSS 优先** | 优先使用官方 API 和 RSS Feed，零反爬风险 |
| **交易所公告源** | 深交所/上交所公告、巨潮资讯 — 最权威、最合规 |
| **爬虫作为最后手段** | 仅在 API/RSS 无法满足需求时启用 |
| **合规约束** | 尊重 robots.txt / 不绕反爬 / 不抓敏感信息 / 低频请求 |
| **白名单制** | 只采集已批准的域名和端点 |

### 11.2 采集源分级

#### Tier 1 — API/RSS (优先，零风险)

| 来源 | 接入方式 | 覆盖率预估 | 实施难度 |
|------|---------|-----------|---------|
| **EastMoney 个股新闻** | `akshare.stock_news_em(symbol)` | 10-30% | ⭐ 低 (一行代码) |
| **Sina 财经 RSS** | `rss.sina.com.cn` → 解析 | 5-15% | ⭐ 低 |
| **深交所公告** | szse.cn API | 3-5% | ⭐⭐ 中 |
| **上交所公告** | sse.com.cn API | 3-5% | ⭐⭐ 中 |
| **巨潮资讯 (已有)** | cninfo.com.cn POST (已接入) | 0.23% | ✅ 已接入 |

#### Tier 2 — 轻量爬虫 (API 不足时启用)

| 来源 | 方式 | robots.txt | 频率限制 | 预估覆盖率 |
|------|------|-----------|---------|-----------|
| **东方财富 7×24** | `requests + BeautifulSoup` | ✅ 允许 | 1 req/5s | 5-10% |
| **同花顺 快讯** | `requests + 解析` | ✅ 允许 | 1 req/5s | 3-8% |
| **雪球 热帖** | `xueqiu.com API` | ✅ 允许 | 1 req/3s | 5-10% |

#### Tier 3 — 禁止项

| 来源 | 原因 |
|------|------|
| 非公开付费数据 | 版权/合规风险 |
| 需要登录/绕过认证 | 违反 ToS |
| 证监会非公开数据 | 法律风险 |
| robots.txt 禁止的路径 | 合规底线 |

### 11.3 爬虫合规约束 (Tier 2 启用时)

```
1. 必须检查目标域名的 robots.txt
2. User-Agent 必须标识为 "AI-Trader/1.0 (research; contact@example.com)"
3. 请求间隔 ≥ 3 秒 (Crawl-Delay)
4. 不解析 JS 渲染页面 (不用 Selenium/Playwright)
5. 不发送登录态 / Cookie
6. 不抓取个人信息、实时行情价格
7. 每个域名每日最多 500 次请求
8. 所有采集请求记录到 data_cache/crawler_log.jsonl
```

### 11.4 建议实施顺序

| 阶段 | 内容 | 预期覆盖率 | 投入 |
|------|------|-----------|------|
| **Phase A** | 接入 EastMoney `stock_news_em` (akshare) | 0.23% → 5-15% | 1-2h |
| **Phase A** | 接入 Sina RSS | +3-8% | 1h |
| **Phase B** | 深交所/上交所公告 API | +3-5% | 2-4h |
| **Phase C** | 轻量爬虫: 东方财富 7×24 | +5-10% | 3-5h |
| **Phase C** | 轻量爬虫: 同花顺快讯 | +3-8% | 2-3h |
| **目标** | 所有 Tier 1 + Tier 2 | **20-40%** | ~15h total |

### 11.5 覆盖率门控自动恢复

当覆盖率逐步提升到阈值以上时，`NewsCoverageGate` 自动恢复资讯因子权重：

```

Phase A 完成: 10% → degraded (×0.3)
Phase C 完成: 25% → normal (×1.0) ← 自动恢复!
```
